# Porting auth, encryption, and accounts

**v2 status: dropped.** v2 has no accounts and no E2EE — onboarding is
"Start fresh" or "Connect GitHub", then land on Today. This doc exists so
nobody rebuilds V1's machinery by reflex, and to name the two ideas from
this area that *do* carry forward.

## What V1 mobile does

- **Sign-in**: Apple and Google (native sheets via
  `@capacitor-firebase/authentication`) or email — OTP code or magic
  link, where magic links round-trip through Firebase Dynamic Links into
  `pages/auth/callback/email.tsx` (`client/screens/auth/`).
- **Token bridge for extensions**: after Firebase auth resolves in the
  webview, the auth-sync plugin (`capacitor/auth-sync.ts`,
  `AuthSyncPlugin.swift`) exchanges the Firebase ID token for Reflect
  OAuth access/refresh tokens via `/api/oauth/token` and stores them —
  plus the current graph ID — in App Group defaults
  (`group.ReflectData`). This is what lets the share extension, recording
  uploads, and widgets authenticate **without the webview**.
- **Graph encryption**: graphs are E2EE; the boot sequence blocks on an
  encryption-unlock gate. Per-graph passwords live in the iOS Keychain
  (`capacitor-secure-storage-plugin`); first-run flows create credentials
  and a **recovery kit** (QR/backup codes the user must save); the
  profile modal can re-create it. Decryption happens on-device before any
  content renders.
- **Account management**: graph switcher, sign out, delete account — all
  in the profile modal. OAuth client credentials and the API endpoint
  ship in a git-ignored `ReflectConfig.plist` generated by a fastlane
  lane.

## What changes in v2, and why

All of it is removed rather than replaced:

- **No accounts.** There is no server to authenticate to. Onboarding is
  Start fresh (create an empty graph in `Documents/`) or Connect GitHub
  (device flow in Safari → token → clone). Sign-out, delete-account, and
  the auth boot gates have no equivalent — do not rebuild the
  loading-gate machinery.
- **No E2EE.** Notes are plain markdown on the user's device; privacy
  comes from local-first storage, `private: true` hard-blocks on cloud
  AI, and transparent egress (GitHub only in mobile v1). No unlock
  screen, no recovery kit, no key ceremony.
- **Secrets still go in the keychain** — the GitHub token (and later AI
  keys) via the same `keyring` module as desktop (`apple-native` covers
  macOS and iOS); never in markdown, git, or `.reflect/`.

## The two ideas that carry forward

1. **Natively-reachable credentials/state for extensions.** V1's App
   Group token store is why capture worked when the webview was dead. v2
   extensions have no tokens to store, but the *pattern* — an App Group
   container as the shared ground between extension targets and the app —
   returns as the capture inbox (audio/share waves), with keychain
   access-group sharing if an extension ever needs a secret.
2. **The unlock lesson in reverse.** V1's most fragile onboarding moments
   were auth + unlock + initial sync stacked in front of the first note.
   v2's acceptance bar (grounding brief §11) is the inversion: installed
   → writing in minutes, fully usable with no account — which is also the
   App Store review story.

## V1 → v2 mapping

| V1                                             | v2                                                        |
| ---------------------------------------------- | ----------------------------------------------------------- |
| Apple/Google/email sign-in                     | Dropped — no accounts                                       |
| Magic links via Dynamic Links                  | Dropped                                                     |
| OAuth token exchange + App Group token store   | Dropped; App Group returns as the capture inbox             |
| Graph encryption + unlock gate + recovery kit  | Dropped — plain markdown, no E2EE                           |
| Per-graph passwords in iOS Keychain            | GitHub token (later AI keys) in iOS Keychain via `keyring`  |
| `ReflectConfig.plist` client credentials       | Gone — no confidential client, nothing to inject            |
| Graph switcher / delete account                | One graph per device in v1; no account to delete            |
